## 2. 잘 된 것 (Keep)

### 2-1. Group K-Fold 검증 전략
- `scenario_id` 기준 그룹 분할로 동일 시나리오 내 타임스텝이 train/valid에 동시 포함되는 leakage를 완전히 차단
- 대회 베이스라인의 KFold 대비 검증 신뢰도가 높아져 실제 리더보드 성능과의 괴리를 줄임

### 2-2. Optuna 하이퍼파라미터 튜닝 (v2)
- LightGBM / XGBoost / CatBoost 세 모델 독립 탐색 (50 trials)
- 튜닝 CV는 3-Fold로 경량화하고 최종 학습은 5-Fold로 진행하는 전략이 유효
- v1 대비 앙상블 RMSE **-0.0814** 개선 달성

### 2-3. Optuna 앙상블 가중치 최적화
- 모델 재학습 없이 OOF만으로 500 trials 탐색 → 수초 내 완료
- RMSE 역수 가중(0.333 균등)보다 최적화된 가중치(LGB 0.35, CAT 0.52)가 더 나은 결과

### 2-4. 어블레이션 테스트 수행
- 피처 추가가 오히려 성능을 낮추는 상황에서 원인을 체계적으로 분리
  - ts 20개 확장 → 큰 폭 악화 확인
  - 타겟 인코딩 비활성화 → 교차 피처 효과 단독 측정
- 섣불리 v3를 제출하지 않고 v2를 최종 채택하는 근거 확보

## 3. 잘 안 된 것 (Problem)

### 3-1. ts_features 확장 (10 → 20개) 역효과
- 추가한 10개 피처(`outbound_truck_wait_min`, `kpi_otd_pct` 등)의 lag/rolling 파생 피처 80개가 오히려 노이즈로 작용
- **근본 원인**: v2 Optuna params가 190개 피처셋에 최적화되어 있었고, 피처가 281개로 늘었을 때 동일 params로는 underfitting 발생 (`n_estimators=700, lr=0.011`로 수렴 부족)
- 피처셋을 변경하면 하이퍼파라미터 재튜닝이 필요하다는 교훈

### 3-2. 타겟 인코딩 효과 미미
- `layout_id`(300개) OOF 타겟 인코딩(mean/std/median)을 추가했으나 단독 기여 불분명
- ts 피처 수 조정 없이 타겟 인코딩만의 순수 효과를 측정하지 못함
- 향후 검증 시 **피처 하나씩 단계적으로 추가**하고 매번 성능 측정하는 방식 필요

### 3-3. 신규 교차 피처 효과 없음
- `robot_efficiency`, `charge_congestion`, `order_pressure`, `ops_stress` 등 8개 추가
- v2 대비 RMSE -0.0068로 사실상 중립 ~ 소폭 악화
- 모델이 이미 원본 피처 조합으로 이 정보를 암묵적으로 학습하고 있을 가능성

## 4. 배운 것 (Learn)

| 교훈 | 내용 |
|------|------|
| **피처 ↑ ≠ 성능 ↑** | 피처 수 증가 시 params 재튜닝이 세트로 필요 |
| **어블레이션은 필수** | 복수의 변경을 한 번에 적용하면 원인 추적 불가 |
| **Optuna params는 피처셋에 종속** | 피처 변경 후 Optuna를 재실행하지 않으면 무효 |
| **앙상블 가중치 최적화는 가성비 최고** | 수초 만에 추가 개선 가능, 항상 포함할 것 |
| **v2 Optuna 결과 저장 필요** | best params를 파일로 저장하지 않아 v3에서 수동 입력 필요했음 |

## 5. 다음 단계 — Phase 3 계획 (Google Colab T4)

### 5-1. 목표
- 로컬 GBDT 앙상블(RMSE 21.2717) 대비 딥러닝 스태킹으로 추가 개선

### 5-2. 구성
```
[1단계] GBDT OOF (v2 결과) → 메타 피처
[2단계] TabNet / MLP 학습 → OOF 생성
[3단계] 모든 OOF → Ridge 메타 모델 → 최종 예측
```

### 5-3. 우선순위 모델
| 모델 | 이유 |
|------|------|
| TabNet | 테이블 데이터 특화, 피처 중요도 해석 가능 |
| MLP (ResNet 스타일) | 빠른 학습, GPU 효율 높음 |
| LSTM | 시계열 구조(24 타임스텝) 명시적 반영 가능 |

### 5-4. 개선 기대 포인트
- 비선형 패턴 포착 (GBDT가 놓친 시계열 장거리 의존성)
- 스태킹으로 다양성 확보 → 앙상블 분산 감소
